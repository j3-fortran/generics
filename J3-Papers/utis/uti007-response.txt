UTI 007 states:

    "Wording improvement requested, but term is unacceptable anyway.

    It is a REALLY bad idea to go around redefining common
    English phrases.

    Also, it is not clear what this is intended to achieve. If it is
    intended to apply everywhere e.g. for image control statements
    being “the same statement”, at a minimum all those need to be
    hunted down and changed into hyperlinks so that at least the
    electronic reader has a clue.

    Actually, it would probably be better if the execution requirement
    were written down in a subclause of templates as a specific
    requirement, and those usages changed to link to that subclause
    (the editor calls those things ”fake terms”). Maybe with a
    cross-reference too."

In reference to new definition:

    3.121 same statement

    the statement that is the one in which the subject appears, and if
    the statement is within a template, is from the same instance of
    the template

Yes, this was an attempt to handle image control statements and
collective subroutines appearing in templates. In addition, it was
realized that a similar problem also  applies to invocation of
statements with "same statement" requirements appearing in generic
subprograms. The suggestion that we do this a better way is a valid
point. Edits are provided.
